# 读书笔记分类器处理规则文档

## 概述
本文档记录了读书笔记分类器的处理规则，确保后续维护时规则一致。

## 笔记分类规则

### 1. 分类标签
- 支持四种分类标签：`#思考`、`#知识`、`#行动`、`#金句`
- 分类标签必须出现在笔记块的开头位置
- 每个笔记块只识别第一个出现的分类标签

### 2. 笔记块识别规则

#### 2.1 个人想法+原文组合笔记
- **开始标记**：`◆` + 日期 + "发表想法"（如：`◆ 2025/09/25发表想法`）
- **处理规则**：
  - 遇到开始标记后，收集后续内容直到遇到"原文："标记
  - 将个人想法内容和原文内容合并为一个完整的笔记块
  - 跳过开始标记行本身（不记录到最终笔记中）

#### 2.2 纯划线笔记
- **标记**：单独的`◆`标记（不是个人想法开始标记）
- **处理规则**：
  - 只记录`◆`标记所在行
  - 后续非标记内容不记录
  - 立即结束当前笔记块

#### 2.3 纯原文笔记
- **标记**：`原文：`
- **处理规则**：
  - 记录`原文：`标记所在行
  - 如果前面有个人想法内容，则合并处理

### 3. 去重规则
- 当`原文：`内容与后续`◆`划线内容完全相同时
- 跳过`◆`划线内容，避免重复记录
- 只保留`原文：`内容

### 4. 章节标题跳过规则
- **触发条件**：连续两个空行
- **跳过内容**：连续两个空行后的第一个非标记内容行
- **跳过时机**：无论当前是否有活跃笔记块都跳过
- **重置条件**：遇到标记内容时重置跳过标记

### 5. 内容过滤规则
- **跳过内容**：
  - 个人想法开始标记行（`◆` + 日期 + "发表想法"）
  - 章节标题（满足跳过条件的非标记内容）
  - 重复的划线内容
- **保留内容**：
  - 分类标签行
  - 个人想法内容
  - 原文内容
  - 纯划线内容（去重后）

### 6. 空白处理
- 笔记块开头和结尾的空白行自动去除
- 中间的空白行保留原样

## 技术实现要点

### 解析状态机
1. **初始状态**：等待分类标签或个人想法开始标记
2. **个人想法状态**：收集个人想法内容，等待原文标记
3. **原文状态**：处理原文内容
4. **纯划线状态**：处理单个划线内容

### 关键变量
- `currentBlock`：当前正在构建的笔记块
- `currentCategory`：当前笔记块的分类
- `consecutiveEmptyLines`：连续空行计数
- `skipNextNonMarked`：跳过下一个非标记内容标记
- `inPersonalNote`：是否处于个人想法收集状态
- `personalNoteContent`：暂存的个人想法内容

## 测试用例示例

### 有效输入示例
```
#知识
所以不要妄想有终点，持续努力

原文：即使逃出了生存时间，也是暂时的，还会进入新的生存时间去忍受煎熬。

◆ 即使逃出了生存时间，也是暂时的，还会进入新的生存时间去忍受煎熬。


如何判断自己是否处于生存时间？

◆ 2025/09/25发表想法
#思考
这个问题很有深度
原文：生存时间的判断标准是...
```

### 预期输出
1. 分类为"知识"的笔记块：
   ```
   #知识
   所以不要妄想有终点，持续努力
   
   原文：即使逃出了生存时间，也是暂时的，还会进入新的生存时间去忍受煎熬。
   ```

2. 分类为"思考"的笔记块（个人想法+原文合并）：
   ```
   #思考
   这个问题很有深度
   原文：生存时间的判断标准是...
   ```

## 维护注意事项

1. **规则一致性**：修改解析逻辑时需确保所有规则仍然有效
2. **测试验证**：每次修改后使用标准测试用例验证功能
3. **日志调试**：保留console.log调试信息便于问题排查
4. **边界情况**：特别注意空行、重复内容、特殊字符等边界情况
